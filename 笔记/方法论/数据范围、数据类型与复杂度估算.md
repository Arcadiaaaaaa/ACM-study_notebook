# 数据范围、数据类型与复杂度估算

比赛里先看数据范围，通常就是先回答两个问题：
- 值会不会溢出，要开哪种数据类型。
- 最坏复杂度能不能在时限内跑完。

## 1. 整数类型怎么选

常用结论：

- `int`：大约到 `2e9`，适合下标、普通计数、题目明确很小的数值。
- `long long`：大约到 `9e18`，适合大多数求和、乘法、答案统计。
- `__int128`：当两个 `long long` 相乘也可能爆掉时再用。

快速判断：

- 单个数在 `1e9` 以内，通常可以先用 `int` 存。
- 但只要出现“求和、乘法、数对数、方案数”，就要额外估答案范围，很多时候答案要开 `long long`。
- 如果有 `a * b`，而 `a`、`b` 都可能到 `1e10`、`1e12`、`1e18` 这种级别，`long long` 也未必够，要考虑 `__int128`。

常见场景：

- `n <= 1e5`，每个数 `<= 1e9`，数组元素本身可以是 `int`，但总和可能到 `1e14`，前缀和、答案要用 `long long`。
- `n <= 1e5`，统计数对个数，最坏是 `n * (n - 1) / 2`，大约 `5e9`，答案必须是 `long long`。
- 二分时如果左右端点可能很大，`mid` 仍然建议写成 `l + (r - l) / 2`。

## 2. 浮点类型怎么选

- `double`：大多数几何、平均值、概率题够用。
- `long double`：精度要求更严时再考虑。

注意：

- 浮点数不要用来做“大整数精确比较”。
- `double` 不是精确整数类型，超过一定范围后会丢失低位。

## 3. 常见空间估算

粗略记忆：

- `int[1e5]` 约 `0.4 MB`
- `int[1e6]` 约 `4 MB`
- `long long[1e6]` 约 `8 MB`
- `bool[1e6]` 约 `1 MB`

所以：

- `1e5` 级数组，随便开。
- `1e6` 级数组，要开始关注类型。
- `1e7` 级数组，通常只能开少量 `int / bool` 数组，`long long` 很容易顶到内存。

## 4. 复杂度大概要控制到多少

这是竞赛里的经验表，不是绝对值，但很实用。

先记一个总原则：

- 单组数据大致按 `1e7 ~ 1e8` 级别基本操作去估。
- 常数大、`STL` 操作多、测试组多时，要比这个更保守。

常见对应关系：

- `n <= 10`：`O(n!)`、`O(2^n)` 往往还能考虑。
- `n <= 20`：`O(2^n)` 常见，`O(n^3)` 也没问题。
- `n <= 200`：`O(n^3)` 通常可做。
- `n <= 2e3`：优先想 `O(n^2)`。
- `n <= 1e4`：`O(n^2)` 一般偏危险，除非常数很小、数据不满；更稳的是 `O(n sqrt n)`、`O(n log n)`。
- `n <= 1e5`：通常要控制在 `O(n log n)` 或 `O(n)`。
- `n <= 1e6`：通常要 `O(n)`，`O(n log n)` 有时能过但要看常数和题目时限。
- `n <= 1e7`：基本只能接受低常数 `O(n)`，还要小心内存。

## 5. 常见复杂度看到范围后的第一反应

- 范围很小，像 `n <= 20`：先想状态压缩、搜索、枚举子集。
- `n <= 200`：先想 `O(n^3)` 的 DP、Floyd、三重枚举。
- `n <= 2000`：先想 `O(n^2)` 枚举、区间 DP、二维转移。
- `n <= 1e5`：先想排序、二分、前缀和、哈希、单调栈、单调队列、树状数组、并查集。
- `n <= 1e6`：先想线性筛、线性 DP、双指针、前缀统计。

## 6. 最容易错的不是“单个数”，而是中间过程

重点检查：

- 数组元素虽然是 `int`，但前缀和、答案可能要 `long long`。
- 乘法比加法更容易爆，比如面积、组合计数、区间贡献。
- 即使最后要 `% mod`，中间乘法也可能先溢出。
- 下标能用 `int`，不代表答案也能用 `int`。

## 7. 比赛里可以直接这样判断

1. 先看单个数值范围，决定数组元素用 `int` 还是 `long long`。
2. 再看有没有求和、乘法、计数、方案数，单独估答案范围。
3. 再看 `n` 的大小，判断自己该选 `O(n^2)`、`O(n log n)` 还是 `O(n)`。
4. 如果题目有多组数据，要把总数据范围一起算进去。

一句话记忆：

- 值域决定“存什么类型”。
- 过程决定“中间会不会爆”。
- 规模决定“复杂度上限”。
