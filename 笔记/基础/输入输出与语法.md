# 输入输出与语法

## 1. 输入输出

### 1.1 C++ 流加速

作用：减少 `cin/cout` 与 `stdio` 同步带来的开销。

```cpp
ios::sync_with_stdio(false);
cin.tie(nullptr);
```

注意：
- 开了加速后，尽量不要再和 `printf/scanf` 混用。
- 如果题目数据量大，优先先写这两句。

### 1.2 控制小数位数

头文件：

```cpp
#include <iomanip>
```

示例：

```cpp
cout << fixed << setprecision(2) << x;
```

说明：
- `fixed`：固定小数形式输出。
- `setprecision(n)`：配合 `fixed` 时表示保留 `n` 位小数。

### 1.3 控制宽度、对齐、填充

```cpp
cout << setw(5) << x;                  // 默认右对齐
cout << left << setw(5) << x;          // 左对齐
cout << setfill('0') << setw(5) << x;  // 不足补 0
```

说明：
- `setw(n)` 只对紧随其后的一次输出生效。
- `setfill(c)` 会持续生效，直到再次修改。

### 1.4 进制输出

```cpp
cout << dec << x;  // 十进制
cout << hex << x;  // 十六进制
cout << oct << x;  // 八进制
```

### 1.5 C 风格格式化输出

头文件：

```cpp
#include <cstdio>
```

示例：

```cpp
printf("%d %.2f\n", a, b);
```

常用格式符：
- `%d`：`int`
- `%lld`：`long long`
- `%.2f`：保留 2 位小数
- `%c`：`char`
- `%s`：字符串

### 1.6 读取字符串

```cpp
cin >> s;         // 读到空格为止
getline(cin, s);  // 读取整行
```

注意：
- 如果前面刚用 `cin` 读完数字，再用 `getline`，要先处理残留换行。
- 文本题只要可能出现空格，就优先考虑 `getline`。

## 2. 常用函数与语法

### 2.1 常用函数

```cpp
abs(x);                          // 绝对值
sqrt(x);                         // 平方根
reverse(s.begin(), s.end());     // 翻转
sort(a.begin(), a.end());        // 排序
tolower(c);                      // 转小写
toupper(c);                      // 转大写
```

常见头文件：
- `abs`：`<cstdlib>` 或 `<cmath>`
- `sqrt`：`<cmath>`
- `reverse`、`sort`：`<algorithm>`
- `toupper/tolower`：`<cctype>`

注意：
- `toupper`、`tolower` 处理的是单个字符。
- 批量统一大小写时，常配合范围 `for` 使用。

### 2.2 开二维 `vector`

```cpp
vector<vector<int>> a(n, vector<int>(m));
```

### 2.3 三目运算符

```cpp
cout << (cond ? x : y);
```

注意：
- 三目运算符优先级较低，复杂表达式建议加括号。

### 2.4 范围 `for`

值遍历：

```cpp
for (char c : s) {
    // c 是副本
}
```

引用遍历：

```cpp
for (char &c : s) {
    // c 是原元素的引用
}
```

常量引用遍历：

```cpp
for (const auto &x : a) {
    // 只读，不拷贝
}
```

说明：
- 值遍历修改 `c` 不会影响原容器。
- 引用遍历修改 `c` 会影响原容器。
- 想改原字符串大小写时，要写成 `char &c`。

### 2.5 作用域

说明：
- 在一个函数内部定义的局部变量，其他函数不能直接访问。
- 如果其他函数要使用，通常有两种做法：
  - 作为参数传入。
  - 定义为全局变量。

比赛里常见取舍：
- 纯模板函数、图论数组、前缀和数组等，常开全局。
- 局部逻辑清晰、数据量不大时，尽量局部定义。
