# 常见技巧与典型思想

## 1. 方向数组

作用：把网格中“相邻位置”的变化量统一存起来，便于循环处理。

八方向：

```cpp
int dx[8] = {-1, -1, -1, 0, 0, 1, 1, 1};
int dy[8] = {-1, 0, 1, -1, 1, -1, 0, 1};
```

示例：

```cpp
for (int k = 0; k < 8; k++) {
    int nx = x + dx[k];
    int ny = y + dy[k];
    if (0 <= nx && nx < n && 0 <= ny && ny < m && isboom(nx, ny)) {
        cnt++;
    }
}
```

注意：
- 使用前先统一坐标含义。
- 越界判断要写全。

## 2. 规则表 / 映射表

适用场景：
- 题目规则本身就是一一对应关系。

示例：

```cpp
beat["elephant"] = "tiger";
if (beat.count(a) && beat[a] == b) {
    // a 能赢 b
}
```

注意：
- 如果 `a` 可能不存在，先判断 `count`，避免误插入默认值。

## 3. 编号后利用取余找规律

适用场景：
- 多个对象构成环形关系。

示例：

```cpp
// elephant = 0, tiger = 1, cat = 2, mouse = 3
// x 吃 y
y == (x + 1) % 4
```

核心思想：
- 先编号。
- 再把题目的循环关系转成模运算。

## 4. 约瑟夫问题

关键思想：
- 删除一个人后，对剩余元素重新编号。
- 把原问题转化为规模减一的同类问题。

本质：
- 下标映射
- 重编号
- 规模缩小

## 5. 日期模拟

适用场景：
- 日期推进
- 计算若干天后的日期
- 判断两个日期之间相隔多少天
- 日历题、签到题、时间模拟题

### 5.1 闰年定义

闰年的定义是：
- 能被 `400` 整除
- 或者能被 `4` 整除但不能被 `100` 整除

写法：

```cpp
bool isLeap(int year) {
    return (year % 400 == 0) || (year % 4 == 0 && year % 100 != 0);
}
```

记忆：
- 普通年份里，2 月有 `28` 天
- 闰年里，2 月有 `29` 天
- 也就是：2 月比平年多一天

### 5.2 每个月多少天

平年可以先写成：

```cpp
int days[13] = {0, 31, 28, 31, 30, 31, 30, 31, 31, 30, 31, 30, 31};
```

如果当前年份是闰年，就把 2 月改成 `29`：

```cpp
days[2] = isLeap(year) ? 29 : 28;
```

### 5.3 常见日期推进写法

例如把日期往后推一天：

```cpp
day++;
if (day > days[month]) {
    day = 1;
    month++;
    if (month > 12) {
        month = 1;
        year++;
    }
}
```

核心顺序：
- 先处理天
- 天越界后处理月
- 月越界后处理年

### 5.4 做题提醒

- 日期题最容易错的是 2 月
- 写月份天数时，先写平年数组，再单独特判闰年
- 只要涉及跨年、跨月，就不要靠脑算，优先老老实实模拟
- 如果题目要求很多次推进日期，优先先把“判断闰年”和“每月天数”写成函数

## 6. 二维前缀和

适用场景：
- 多次查询某个子矩阵元素和
- 矩阵固定不变，但查询很多
- 想把每次 `O(nm)` 的区域求和降到 `O(1)`

核心思想：
- 一维前缀和是“前 `i` 个数的和”
- 二维前缀和是“从左上角 `(1,1)` 到 `(i,j)` 的整个矩形和”

通常定义：

```cpp
s[i][j] = a[1][1] 到 a[i][j] 这个矩形的元素和
```

递推公式：

```cpp
s[i][j] = s[i - 1][j] + s[i][j - 1] - s[i - 1][j - 1] + a[i][j];
```

为什么要减掉 `s[i - 1][j - 1]`：
- 因为它在前两项里被重复算了一次

模板：

```cpp
for (int i = 1; i <= n; i++) {
    for (int j = 1; j <= m; j++) {
        s[i][j] = s[i - 1][j] + s[i][j - 1] - s[i - 1][j - 1] + a[i][j];
    }
}
```

查询子矩阵 `(x1, y1)` 到 `(x2, y2)` 的和：

```cpp
sum = s[x2][y2] - s[x1 - 1][y2] - s[x2][y1 - 1] + s[x1 - 1][y1 - 1];
```

理解方式：
- 先取大矩形 `s[x2][y2]`
- 减去上面多算的一块
- 减去左边多算的一块
- 左上角被减重了，再补回来一次

做题提醒：
- 最好统一用 1 下标，这样公式最整齐
- 数组常开成 `n + 1` 行、`m + 1` 列，方便处理第 `0` 行和第 `0` 列
- 先统一坐标含义：通常 `i` 表示行，`j` 表示列
- 如果矩阵值大、矩形和也可能很大，前缀和要开 `long long`

## 7. 二维差分

适用场景：
- 多次给某个子矩阵整体加上一个值
- 修改很多，最后再统一求结果
- 想把每次区域修改从 `O(nm)` 降到 `O(1)`

核心思想：
- 一维差分是“记录变化的起点和终点”
- 二维差分是“记录一个矩形对四个角的影响”

如果要给矩形 `(x1, y1)` 到 `(x2, y2)` 全部加上 `c`，则：

```cpp
b[x1][y1] += c;
b[x2 + 1][y1] -= c;
b[x1][y2 + 1] -= c;
b[x2 + 1][y2 + 1] += c;
```

含义是：
- 从左上角开始加
- 在矩形下方消掉影响
- 在矩形右侧消掉影响
- 在右下角把重复消掉的部分补回来

常用封装：

```cpp
void insert(int x1, int y1, int x2, int y2, int c) {
    b[x1][y1] += c;
    b[x2 + 1][y1] -= c;
    b[x1][y2 + 1] -= c;
    b[x2 + 1][y2 + 1] += c;
}
```

所有修改做完后，再还原最终矩阵：

```cpp
for (int i = 1; i <= n; i++) {
    for (int j = 1; j <= m; j++) {
        b[i][j] += b[i - 1][j] + b[i][j - 1] - b[i - 1][j - 1];
    }
}
```

这里还原时用的，其实也是二维前缀和思想。

如果原矩阵一开始就有值：
- 可以把每个 `a[i][j]` 看成对矩形 `(i, j, i, j)` 加上 `a[i][j]`
- 也就是先对每个点做一次 `insert(i, j, i, j, a[i][j])`
- 然后再处理题目给出的矩形修改

做题提醒：
- 仍然建议统一用 1 下标
- 差分数组通常要多开一圈，避免访问 `x2 + 1`、`y2 + 1` 越界
- 如果题目只改不查，差分通常比每次暴力修改高效得多
- 如果题目既频繁修改又频繁查询，就不能只靠普通前缀和 / 差分，往往要考虑树状数组、线段树一类结构

## 8. 轮次标记法：用 `tag` 代替反复清空 `vis`

适用场景：
- 外层一轮一轮枚举左端点
- 每一轮都要重新统计“这个值在本轮是否出现过”
- 如果每轮都 `memset(vis, 0, sizeof vis)`，会产生很多重复开销

典型原型是：
- 枚举左端点 `l`
- 向右枚举右端点 `r`
- 需要判断 `a[r]` 在这一轮里是不是第一次出现

朴素想法通常是：

```cpp
for (int l = 1; l <= n; l++) {
    memset(vis, 0, sizeof vis);
    for (int r = l; r <= n; r++) {
        if (!vis[a[r]]) {
            vis[a[r]] = 1;
            // 第一次出现
        }
    }
}
```

问题在于：
- 逻辑上每一轮确实要“清空”
- 但不一定真的要把整个数组重新赋成 `0`

优化思想：
- 给每一轮枚举分配一个唯一编号 `tag`
- `vis[x]` 不再只存 `0 / 1`
- 而是存“这个值上一次在哪一轮出现过”

于是：
- 如果 `vis[x] == tag`，说明 `x` 在当前这一轮已经出现过
- 如果 `vis[x] != tag`，说明 `x` 在当前这一轮还没出现过

典型写法：

```cpp
int tag = 0;
for (int l = 1; l <= n; l++) {
    tag++;
    for (int r = l; r <= n; r++) {
        int x = a[r];
        if (vis[x] == tag) continue;
        vis[x] = tag;
        // x 在这一轮第一次出现
    }
}
```

这句的本质是：

```text
vis[x] == tag
```

不表示“这个值永远出现过”，而表示：

```text
这个值在当前轮次里已经出现过
```

为什么这样能省掉清空数组：
- 因为旧轮次留下来的标记，轮次编号已经不同了
- 只要 `tag` 变了，旧数据就自动“失效”
- 相当于用“当前轮次编号”把不同轮的访问记录隔开了

这类优化的本质：
- 不是清空状态
- 而是给状态打上“属于哪一轮”的时间戳

常见场景：
- 枚举每个左端点，统计不同元素个数
- 枚举区间起点时，对当前轮做去重
- 多次独立搜索 / 多次独立统计时，避免反复清空标记数组

什么时候特别值得用：
- `vis` 数组很大
- 外层轮数很多
- 每轮真正访问到的值远少于数组大小

使用前提：
- 值 `x` 能直接当数组下标，或者已经离散化
- 如果值域特别大且稀疏，可能更适合 `unordered_map`

做题提醒：
- `tag` 每轮都要加一，不能忘
- 如果轮数特别多，要注意 `tag` 的类型，通常用 `int` 就够，大到担心溢出时可改 `long long`
- 这种写法优化的是“清空数组”的代价，不会改变内层真正访问元素的复杂度

记忆方式：
- `vis[x]` 存的不是“是否访问过”
- 而是“上次在哪一轮访问过”
