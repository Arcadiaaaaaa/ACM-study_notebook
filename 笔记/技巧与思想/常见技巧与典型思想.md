# 常见技巧与典型思想

## 1. 方向数组

作用：把网格中“相邻位置”的变化量统一存起来，便于循环处理。

八方向：

```cpp
int dx[8] = {-1, -1, -1, 0, 0, 1, 1, 1};
int dy[8] = {-1, 0, 1, -1, 1, -1, 0, 1};
```

示例：

```cpp
for (int k = 0; k < 8; k++) {
    int nx = x + dx[k];
    int ny = y + dy[k];
    if (0 <= nx && nx < n && 0 <= ny && ny < m && isboom(nx, ny)) {
        cnt++;
    }
}
```

注意：
- 使用前先统一坐标含义。
- 越界判断要写全。

## 2. 规则表 / 映射表

适用场景：
- 题目规则本身就是一一对应关系。

示例：

```cpp
beat["elephant"] = "tiger";
if (beat.count(a) && beat[a] == b) {
    // a 能赢 b
}
```

注意：
- 如果 `a` 可能不存在，先判断 `count`，避免误插入默认值。

## 3. 编号后利用取余找规律

适用场景：
- 多个对象构成环形关系。

示例：

```cpp
// elephant = 0, tiger = 1, cat = 2, mouse = 3
// x 吃 y
y == (x + 1) % 4
```

核心思想：
- 先编号。
- 再把题目的循环关系转成模运算。

## 4. 约瑟夫问题

关键思想：
- 删除一个人后，对剩余元素重新编号。
- 把原问题转化为规模减一的同类问题。

本质：
- 下标映射
- 重编号
- 规模缩小

## 5. 日期模拟

适用场景：
- 日期推进
- 计算若干天后的日期
- 判断两个日期之间相隔多少天
- 日历题、签到题、时间模拟题

### 5.1 闰年定义

闰年的定义是：
- 能被 `400` 整除
- 或者能被 `4` 整除但不能被 `100` 整除

写法：

```cpp
bool isLeap(int year) {
    return (year % 400 == 0) || (year % 4 == 0 && year % 100 != 0);
}
```

记忆：
- 普通年份里，2 月有 `28` 天
- 闰年里，2 月有 `29` 天
- 也就是：2 月比平年多一天

### 5.2 每个月多少天

平年可以先写成：

```cpp
int days[13] = {0, 31, 28, 31, 30, 31, 30, 31, 31, 30, 31, 30, 31};
```

如果当前年份是闰年，就把 2 月改成 `29`：

```cpp
days[2] = isLeap(year) ? 29 : 28;
```

### 5.3 常见日期推进写法

例如把日期往后推一天：

```cpp
day++;
if (day > days[month]) {
    day = 1;
    month++;
    if (month > 12) {
        month = 1;
        year++;
    }
}
```

核心顺序：
- 先处理天
- 天越界后处理月
- 月越界后处理年

### 5.4 做题提醒

- 日期题最容易错的是 2 月
- 写月份天数时，先写平年数组，再单独特判闰年
- 只要涉及跨年、跨月，就不要靠脑算，优先老老实实模拟
- 如果题目要求很多次推进日期，优先先把“判断闰年”和“每月天数”写成函数
