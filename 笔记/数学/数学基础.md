# 数学基础

## 1. 整数反转

```cpp
long long t = x, b = 0;
while (t) {
    b = b * 10 + (t % 10);
    t /= 10;
}
```

注意：
- 数据范围大时，要考虑反转过程中是否溢出。
- 若 `x = 0`，要注意循环是否还能得到正确结果。

## 2. 最大公约数 `gcd`

```cpp
long long gcd(long long a, long long b) {
    return b == 0 ? a : gcd(b, a % b);
}
```

## 3. 三角形判定

若三边为 `a, b, c`，构成三角形的条件是：
- `a > 0`，`b > 0`，`c > 0`
- 任意两边之和大于第三边

若已知 `a <= b <= c`，只需判断：

```cpp
a + b > c
```

注意：
- 数据范围大时，`a + b` 可能溢出，必要时使用 `long long`。

## 4. 模运算

下列运算中，可以边算边取模：
- 加法
- 减法
- 乘法

常用性质：

```cpp
(a + b) % mod = ((a % mod) + (b % mod)) % mod
(a - b) % mod = ((a % mod) - (b % mod) + mod) % mod
(a * b) % mod = ((a % mod) * (b % mod)) % mod
```

注意：
- 除法不能直接写成“先除再取模”。
- 模意义下做除法，通常要乘逆元，而且有前提条件。
- 即使最终会取模，中间过程也可能先溢出。

## 5. 取余、取模、负数修正

在 C++ 中，`%` 是取余，不是数学意义上的“总是非负”的取模。

例如：

```cpp
-2 % 26 == -2
```

因此常见修正写法是：

```cpp
(x % m + m) % m
```

作用：
- 把结果规范到 `0 ~ m - 1` 范围内。

例子：

```cpp
(c - 'a' - n) % 26              // 可能错
((c - 'a' - n) % 26 + 26) % 26  // 更稳
```

## 6. 阶乘与 `10^k` 的倍数

阶乘会把很多连续整数乘起来，因此会不断累积出很多个因子 `2` 和 `5`。

因为：

```text
10 = 2 × 5
10^k = 2^k × 5^k
```

所以一个数想被 `10^k` 整除，只要它里面至少有 `k` 个因子 `2` 和 `k` 个因子 `5`。

这说明：
- 每凑出一对 `2` 和 `5`，末尾就会多一个 `0`
- 一个数末尾有多少个 `0`，本质上就是看它能分解出多少组 `2 × 5`

对阶乘来说，因子 `2` 的数量通常比因子 `5` 多得多，所以末尾 `0` 的个数实际上由因子 `5` 的个数决定。

因此，`n!` 末尾 `0` 的个数为：

```text
⌊n / 5⌋ + ⌊n / 25⌋ + ⌊n / 125⌋ + ...
```

原因：
- 每个 `5` 贡献一个因子 `5`
- 每个 `25` 额外多贡献一个因子 `5`
- 每个 `125` 再额外多贡献一个因子 `5`
- 以此类推

做题提醒：
- 遇到很大的阶乘时，不要真的把整个阶乘算出来
- 优先考虑它是否是 `10^k` 的倍数，能不能直接用质因子个数化简
- 这类题常常是在问末尾有几个 `0`、能被多少个 `10` 整除、最后几位是否为 `0`

## 7. 连续阶乘的增量维护

像 `1! × 2! × 3! × ... × n!` 这种题，不能把每个阶乘都从头算一遍，否则会有大量重复计算。

核心想法：
- 阶乘末尾 `0` 的个数，本质上看因子 `5` 的个数
- 从 `(i - 1)!` 变到 `i!` 时，只是多乘了一个 `i`
- 所以 `i!` 中因子 `5` 的个数，可以在前一个结果基础上增量更新

记：

```text
f(i) = i! 中因子 5 的个数
cnt5(i) = i 中因子 5 的个数
```

那么有：

```text
f(i) = f(i - 1) + cnt5(i)
```

其中：

```cpp
int cnt5(int x) {
    int res = 0;
    while (x % 5 == 0) {
        res++;
        x /= 5;
    }
    return res;
}
```

如果题目要求的是：

```text
1! × 2! × 3! × ... × n!
```

末尾有多少个 `0`，那么答案就是：

```text
f(1) + f(2) + f(3) + ... + f(n)
```

做题提醒：
- 连续阶乘题，优先想“前一个结果能不能复用”
- 不要每次重新计算 `i!`
- 常见写法是用一个变量维护当前 `i!` 里因子 `5` 的个数，再不断累加到答案里

这类题的关键优化不是高精度，而是发现“阶乘之间只有一步增量”。

## 8. 同余

如果

```cpp
a ≡ b (mod m)
c ≡ d (mod m)
```

那么有：

```cpp
a + c ≡ b + d (mod m)
a - c ≡ b - d (mod m)
a * c ≡ b * d (mod m)
```

结论：
- 加、减、乘都能把数先化到模意义下再计算。
- 这也是很多计数题、循环节题、密码题的基础。

例题：

```text
20^22 % 7 = 1
```

思路：

```text
20 ≡ 6 ≡ -1 (mod 7)
```

所以：

```text
20^22 ≡ (-1)^22 ≡ 1 (mod 7)
```

因此：

```text
20^22 % 7 = 1
```

这类题的关键不是先把大数真的算出来，而是先把底数在模意义下化简。

## 9. 数根与模 9

一个数与它的各位数字和，对 `9` 取模的结果相同。

例如：

```text
1234 % 9 = (1 + 2 + 3 + 4) % 9
```

原因：
- 十进制下

```text
1234 = 1 × 10^3 + 2 × 10^2 + 3 × 10 + 4
```

- 而

```text
10 ≡ 1 (mod 9)
```

- 所以

```text
10^k ≡ 1 (mod 9)
```

于是原式在模 `9` 意义下就变成：

```text
1 + 2 + 3 + 4
```

结论：
- 一个数和它的各位数字和模 `9` 同余
- 不断把各位数字相加，最终得到的结果和原数模 `9` 同余

常见用途：
- 判断能否被 `9` 整除
- 数根问题
- 数位和相关题目

判断能否被 `9` 整除：
- 一个数能被 `9` 整除，当且仅当它的各位数字和能被 `9` 整除

数根：
- 不断把各位数字相加，直到剩下一位数
- 这个结果叫数根

除了 `0` 以外：

```text
dr(x) = (x - 1) % 9 + 1
```

如果 `x = 0`，数根是 `0`。

## 10. 进制转换

### 10.1 十进制转 `k` 进制

核心方法：
- 不断对 `k` 取余
- 余数就是当前最低位
- 再把原数除以 `k`
- 最后把得到的各位倒过来

模板：

```cpp
string decimalToBase(long long n, int base) {
    if (n == 0) return "0";

    string s;
    while (n > 0) {
        int rem = n % base;
        if (rem < 10) s += char('0' + rem);
        else s += char('A' + rem - 10);
        n /= base;
    }
    reverse(s.begin(), s.end());
    return s;
}
```

记忆：
- 取余得到的是低位到高位
- 所以最后一定要 `reverse`

### 10.2 `k` 进制转十进制

核心方法：
- 从左到右扫描每一位
- 当前结果不断乘上 `base`
- 再加上当前这一位的数值

模板：

```cpp
long long baseToDecimal(const string &s, int base) {
    long long res = 0;
    for (char c : s) {
        int val;
        if ('0' <= c && c <= '9') val = c - '0';
        else val = c - 'A' + 10;
        res = res * base + val;
    }
    return res;
}
```

记忆：

```text
res = res * base + 当前位
```

这和“把一个十进制数按位展开”的思路本质一样。

### 10.3 常见字符映射

当进制大于 10 时：
- `0 ~ 9` 表示 `0 ~ 9`
- `A ~ Z` 表示 `10 ~ 35`

例如十六进制：
- `A = 10`
- `B = 11`
- `C = 12`
- `D = 13`
- `E = 14`
- `F = 15`

### 10.4 常见题型

- 十进制转二进制 / 八进制 / 十六进制
- 把一个 `k` 进制字符串转成十进制
- 判断一个数在不同进制下的表示形式
- 数位模拟题、回文进制题、进制加法题

### 10.5 做题提醒

- `0` 要单独处理，否则循环可能直接跳过
- 十进制转 `k` 进制后别忘了翻转
- 大于 10 的进制要处理好字母和数字的映射
- 如果题目数据很大，要注意转回十进制时可能溢出
- 有负数时，通常先处理符号，再处理绝对值

## 11. 二进制位运算

常见运算符：

```cpp
a & b   // 按位与
a | b   // 按位或
a ^ b   // 按位异或
~a      // 按位取反
a << k  // 左移 k 位
a >> k  // 右移 k 位
```

### 11.1 常见含义

- `a & b`：两个位置都为 `1`，结果才是 `1`
- `a | b`：只要有一个位置为 `1`，结果就是 `1`
- `a ^ b`：两个位置不同，结果才是 `1`
- `a << k`：通常相当于乘 `2^k`
- `a >> k`：通常相当于除以 `2^k`

### 11.2 常见写法

判断奇偶：

```cpp
if (x & 1) { ... }   // 奇数
```

取第 `k` 位：

```cpp
int bit = (x >> k) & 1;
```

把第 `k` 位置为 `1`：

```cpp
x |= (1 << k);
```

把第 `k` 位清成 `0`：

```cpp
x &= ~(1 << k);
```

翻转第 `k` 位：

```cpp
x ^= (1 << k);
```

### 11.3 `lowbit`

作用：
- 取出二进制下最低位的 `1`

写法：

```cpp
int lowbit(int x) {
    return x & -x;
}
```

例如：

```text
x = 12 = 1100(2)
lowbit(x) = 4 = 0100(2)
```

常见用途：
- 树状数组
- 拆二进制最低位

### 11.4 位掩码常见场景

用一个整数表示一个集合：
- 第 `k` 位是 `1`，表示第 `k` 个元素被选了
- 第 `k` 位是 `0`，表示没选

判断第 `k` 个元素是否选中：

```cpp
if ((mask >> k) & 1) { ... }
```

枚举 `n` 个元素的所有子集：

```cpp
for (int mask = 0; mask < (1 << n); mask++) {
    ...
}
```

### 11.5 做题提醒

- 位运算优先级不高，复杂表达式加括号更稳
- `1 << k` 里如果 `k` 很大，注意是否要写成 `1LL << k`
- 用位运算表示集合时，要先想清楚第几位对应哪个元素
- `^` 是异或，不是乘方

## 12. 点到直线距离

若点 `A`、`B` 确定一条直线，点 `P` 到该直线的距离为：

```cpp
d = |AB x AP| / |AB|
```

二维向量叉乘：

```cpp
(x1, y1) x (x2, y2) = x1 * y2 - y1 * x2
```

注意：
- 分子要取绝对值。
- 这是“点到直线”的距离，不是“点到线段”的距离。
