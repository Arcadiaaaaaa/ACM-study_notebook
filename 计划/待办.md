> 目标：按补齐短板的收益排序推进，优先补最影响比赛表现的内容。

## 第一档：最优先补

### 1. 二分答案

- [ ] 对应题：H
- [ ] 当前判断：已经会一半了，最容易补起来

- [ ] 最大化最小值 / 最小化最大值
- [ ] `check(mid)` 的单调性判断
- [ ] 两种二分模板
- [ ] `long long` 边界

### 2. 位运算与二进制

- [ ] 对应题：F
- [ ] 当前判断：这块明显不熟

练什么：

- [ ] 二进制表示

### 3. 图论最短路

- [ ] 对应题：I

练什么：

- [ ] 邻接表

- [ ] 堆优化
- [ ] 多源 / 多次最短路后的枚举

### 4. 复杂度优化意识

- [ ] 对应题：E

- [ ] 从 `O(n^3)` 降到 `O(n^2)` / `O(n log n)` / `O(n)`

## 第三档：顺手改掉的坏习惯

> 这些不需要单独开专题，但每次写代码都要主动检查。

### 7. 边界和特殊值

- [ ] 对应题：A、F、H

重点：

- [ ] `n = 0`
- [ ] `i / j` 会不会越界
- [ ] 重复值是否一定存在
- [ ] 答案是否可能为 `0`

### 8. 数据类型

- [ ] 对应题：H、J

重点：

- [ ] 求和、计数、乘法优先想 `long long`
- [ ] 一看到 `1e5 * 1e9` 量级就警觉

### 9. 代码规范

- [ ] 对应题：D 等

重点：

- [ ] 不混用 `cin` 和 `scanf`
- [ ] 模板写熟
- [ ] 变量名别太随意
